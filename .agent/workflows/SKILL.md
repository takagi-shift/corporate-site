# コーポレートサイト自動生成スキル

## 概要

フォルダ内にある会社情報ファイル（`.docx`）を読み取り、モダンで洗練されたコーポレートサイトを HTML / CSS / JS の3ファイルで生成するワークフロー。

---

## 前提条件

- 作業フォルダ内に会社情報を含むファイル（`.docx`）が存在すること
- docx ファイルには以下の情報が含まれていることが望ましい：
  - 会社名、代表者、設立、資本金、従業員数、所在地、連絡先
  - 事業内容、対応領域
  - ミッション / ビジョン / バリュー
  - 実績、導入事例
  - 料金

---

## ワークフロー手順

### Step 1: 会社情報ファイルの読み取り

1. 作業フォルダ内の `.docx` ファイルを探す
2. スクリプト `scripts/extract-docx.ps1` を使って docx からテキストを抽出する
   ```powershell
   # 使い方
   powershell -ExecutionPolicy Bypass -File .agent/workflows/scripts/extract-docx.ps1 -DocxPath "会社情報.docx" -OutputPath "extracted.txt"
   ```
3. 抽出したテキストから会社情報を構造化して整理する

// turbo

### Step 2: ユーザーへの実装計画確認

以下の項目をユーザーに提示し、承認を得る：

- **デザイン方針**: ダークネイビー × 電気ブルー、グラスモーフィズム
- **ページ構成**: セクション一覧（ヒーロー、事業内容、実績、事例、料金、会社概要、お問い合わせ等）
- **ファイル構成**: `index.html`, `style.css`, `main.js`
- **機能**: スクロールアニメーション、カウントアップ、レスポンシブ対応

ユーザーの承認（`y`）が得られてから次のステップに進む。

### Step 3: CSS の作成（style.css）

スクリプト `scripts/generate-css.ps1` でテンプレートを生成するか、手動で以下を含むCSSを作成する：

1. **デザイントークン**（CSS カスタムプロパティ）
   - カラーパレット: `--color-bg-primary: #0A1628`, `--color-accent-primary: #4A90E2` 等
   - フォント: `Noto Sans JP` + `Outfit`（Google Fonts）
   - スペーシング、ボーダー、シャドウ
2. **ベーススタイル**: リセット、コンテナ、セクション共通
3. **コンポーネントスタイル**: ヘッダー、ヒーロー、カード、タイムライン、テーブル、フォーム、フッター
4. **アニメーション**: フェードイン（`.fade-in`）
5. **レスポンシブ対応**: 1024px（タブレット）、768px（スマホ）のブレークポイント

### Step 4: HTML の作成（index.html）

以下のセクション構成で作成する：

| # | セクション | 内容 |
|---|:---|:---|
| 1 | ヘッダー | ロゴ + ナビ + CTAボタン + モバイルメニュー |
| 2 | ヒーロー | キャッチコピー + CTA + 実績サマリ |
| 3 | 事業内容 | 対応領域をカード形式で表示 |
| 4 | 支援の進め方 | ステップ形式のタイムライン |
| 5 | 実績 | 数値カウントアップ + 成果ハイライト |
| 6 | 導入事例 | ケースカード（課題→支援→成果） |
| 7 | 料金 | プランカード + 目安価格 |
| 8 | MVV | ミッション / ビジョン / バリュー |
| 9 | 代表メッセージ | プロフィール + メッセージ文 |
| 10 | 会社概要 | テーブル形式 |
| 11 | お問い合わせ | フォーム（ダミー送信） + 連絡先 |
| 12 | フッター | コピーライト + リンク集 |

**重要ポイント:**
- 全要素に `.fade-in` クラスを付与（アニメーション用）
- 数値には `data-count` 属性を付与（カウントアップ用）
- セマンティックHTML（`section`, `nav`, `header`, `footer`）を使用
- SEO対策（meta description, h1構造）

### Step 5: JavaScript の作成（main.js）

以下の機能を実装する：

1. **ヘッダースクロール追従**: スクロール時に背景を半透明＋ブラー
2. **モバイルメニュー開閉**: ハンバーガーメニューのトグル
3. **フェードインアニメーション**: Intersection Observer で `.fade-in` 要素を監視
4. **カウントアップアニメーション**: `data-count` を持つ要素の数値をアニメーション
5. **スムーズスクロール**: アンカーリンクのスムーズスクロール
6. **フォーム送信処理**: ダミー送信（submitイベントをキャンセルして成功メッセージ表示）

### Step 6: 動作確認

1. `index.html` をブラウザで開く
2. 各セクションの表示を確認
3. スクロールアニメーションの動作を確認
4. モバイル表示（375px）でレイアウト崩れがないか確認
5. ナビリンクが正しいセクションに飛ぶか確認

### Step 7: クリーンアップ

- docx 展開用の一時フォルダ（`docx_temp/`）を削除
- 不要な中間ファイルを削除

## デザインカスタマイズガイド

### カラー変更

`style.css` の `:root` 内のCSS変数を変更するだけで全体のテーマを変更可能：

```css
:root {
  --color-bg-primary: #0A1628;      /* 背景メイン */
  --color-accent-primary: #4A90E2;  /* アクセントカラー */
  --color-accent-secondary: #38BDF8; /* アクセントサブ */
}
```

### ライトモード化

背景色を白系、テキストを濃色に変更：

```css
:root {
  --color-bg-primary: #FFFFFF;
  --color-bg-secondary: #F8FAFC;
  --color-text-primary: #1E293B;
  --color-text-secondary: #475569;
}
```

## 使用技術

| カテゴリ | 技術 |
|:---|:---|
| マークアップ | HTML5（セマンティック） |
| スタイル | Vanilla CSS（カスタムプロパティ、Grid、Flexbox） |
| スクリプト | Vanilla JavaScript（Intersection Observer、requestAnimationFrame） |
| フォント | Google Fonts（Noto Sans JP, Outfit） |
| デザイン | グラスモーフィズム、ダークモード、グラデーション |
